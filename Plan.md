# Описание задачи
Создать современный, производительный загрузчик модов (Mod Loader) для игры Slay the Spire 1. Приложение должно стать легковесной альтернативой стандартному ModTheSpire, предоставляя аналогичный двухпанельный интерфейс, но со строгим современным дизайном в темной теме.

Приложение должно быть полностью написано на Rust с использованием GUI-фреймворка Iced и обладать строгой модульной архитектурой для обеспечения чистоты кода, легкого тестирования и возможности расширения.

Интерфейс должен состоять из:
* **Левой панели**: список найденных модов с чекбоксами активации, строкой поиска/фильтрации.
* **Правой панели**: детальная информация о выбранном моде (описание, авторы, версии, зависимости).
* **Блока управления (над списком модов)**:
    1. Кнопка обновления списка модов (повторное сканирование локальной директории).
    2. Кнопка включения всех отображаемых модов.
    3. Кнопки "+" (сохранить текущий набор включенных модов как новый профиль) и "-" (удалить выбранный профиль).
    4. Выпадающий список (PickList) для быстрого переключения между сохраненными профилями.
* **Нижней панели**: кнопка "Play" для запуска игры и чекбокс включения режима Debug.

# Модульная архитектура
Проект необходимо разделить на независимые слабосвязанные модули (с использованием структуры workspaces или четкого разделения по crates/modules):
* `sts_modloader_core`: общая шина событий, управление конфигурацией и инициализация.
* `sts_modloader_fs`: сканирование директорий, валидация путей Steam, асинхронное чтение файлов и запуск процесса Java.
* `sts_modloader_parser`: парсинг метаданных `ModTheSpire.json` из `.jar` архивов.
* `sts_modloader_profile`: логика создания, удаления, импорта и экспорта профилей конфигурации модов.
* `sts_modloader_ui`: компоненты интерфейса, управление состоянием (Application/Sandbox в Iced), кастомные виджеты и стили темной темы.

# Список библиотек
* `iced` - основной фреймворк для построения native GUI на Rust по архитектуре Elm.
* `serde` и `serde_json` - парсинг `ModTheSpire.json` и сохранение профилей пользователя.
* `walkdir` - рекурсивный поиск файлов модов в директориях.
* `zip` - чтение `.jar` файлов без их полной распаковки для получения информации о моде.
* `tokio` - асинхронный рантайм для выполнения тяжелых задач (сканирование, запуск игры) в фоне без фризов UI.

# Что не делать
* Не использовать веб-технологии. Никакого HTML/CSS, Tauri, Electron или системных WebView. Весь интерфейс должен рендериться нативно через Iced (WGPU/Tiny-skia).
* Не использовать светлую тему. Применить встроенную темную тему Iced (`Theme::Dark`) или создать ее кастомный аналог. Цвета должны быть зафиксированы на темной палитре.
* Не смешивать логику UI и работу с файлами. Все операции с диском и запуск процессов должны быть вынесены в отдельные модули (`fs`, `parser`) и вызываться из UI только через механизм `Command` в Iced.
* Не хардкодить путь к папке Slay the Spire. Реализовать логику автоматического поиска через пути Steam (поиск файла `libraryfolders.vdf`), а при неудаче выводить нативное диалоговое окно для ручного выбора папки.
* Не распаковывать `.jar` файлы на диск. Использовать библиотеку `zip` только для точечного потокового чтения файла метаданных внутри архива.
* Не скрывать ошибки запуска. Если процесс Java завершился с ошибкой, модуль `fs` должен перехватить `stderr` и передать его через сообщение (Message) в UI-модуль для вывода пользователю.
